## 🎯 从Windows到Mac的完整流程

### 方案一：使用Git（推荐）⭐

这是最专业的方式，也方便后续协作和版本管理。

#### Step 1: 在Windows上初始化Git仓库

```bash
# 在项目根目录（d:\GameRec）
cd d:\GameRec

# 初始化Git仓库
git init

# 创建.gitignore文件（忽略不需要的文件）
# 见下方.gitignore内容

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit: 完成数据层、存储层、推荐算法和UI示例"
```

#### Step 2: 推送到GitHub

```bash
# 在GitHub上创建新仓库（GameRec）

# 关联远程仓库
git remote add origin https://github.com/你的用户名/GameRec.git

# 推送到GitHub
git push -u origin main
```

#### Step 3: 在Mac上克隆项目

```bash
# 在Mac的终端
cd ~/Documents

# 克隆项目
git clone https://github.com/你的用户名/GameRec.git

# 进入项目目录
cd GameRec
```

#### Step 4: 在Mac上使用Xcode打开

1. **方式A**：双击 `GameRec.xcodeproj`（需要先创建）
2. **方式B**：在终端运行 `open GameRec.xcodeproj`
3. **方式C**：打开Xcode，File → Open → 选择项目文件夹

1. 在Windows上压缩整个 `d:\GameRec` 文件夹

## 🔧 在Mac上首次设置Xcode工程

### 重要：Xcode工程文件需要在Mac上创建

由于 `.xcodeproj` 是二进制格式，无法在Windows上创建。按照以下步骤：

### Step 1: 创建Xcode工程

1. 打开Xcode
2. File → New → Project
3. 选择：iOS → App
4. 填写信息：
   - Product Name: `GameRec`
   - Team: 你的Apple ID
   - Organization Identifier: `com.yourname`
   - Interface: **SwiftUI**
   - Language: **Swift**
   - Storage: **None**
   - Include Tests: ✅
5. 保存位置：选择克隆下来的 `GameRec` 目录

### Step 2: 添加现有文件到工程

在Xcode的Project Navigator中：

1. **删除默认生成的文件**：
   - ContentView.swift（我们有自己的）
   - 保留 GameRecApp.swift 和 Assets.xcassets

2. **添加我们的文件**：
   - 右键 `GameRec` group → Add Files to "GameRec"
   - 选择所有 `.swift` 文件所在的目录
   - ⚠️ 勾选 "Create groups"
   - ⚠️ 勾选 "Add to targets: GameRec"

### Step 3: 添加种子数据JSON

1. 在Xcode中创建 `Data/SeedData` group
2. Add Files → 选择3个JSON文件
3. ⚠️ **不要勾选** "Copy items if needed"（已在正确位置）
4. ⚠️ 勾选 "Add to targets: GameRec"

### Step 4: 设置项目配置

1. 选择项目根节点（蓝色图标）
2. 选择 GameRec target
3. General 标签：
   - **Minimum Deployments**: iOS 15.0
   - **Supported Destinations**: iPhone
4. Build Settings 标签：
   - 确保 Swift Language Version: Swift 5

### Step 5: 修改App入口文件

编辑 `GameRecApp.swift`：

```swift
import SwiftUI

@main
struct GameRecApp: App {
    
    init() {
        // 初始化用户数据（首次启动）
        DataStore.shared.initializeUserDataIfNeeded()
    }
    
    var body: some Scene {
        WindowGroup {
            HomeView()
        }
    }
}
```

## 🚀 运行和测试

### 在模拟器运行

1. 选择目标：顶部工具栏选择 `GameRec` → `iPhone 15`
2. 点击 ▶️ 运行按钮（或 ⌘R）
3. 等待编译和启动

### 在真机运行（iPhone SE3）

1. 用USB连接iPhone到Mac
2. 在iPhone上：设置 → 通用 → VPN与设备管理 → 信任开发者
3. Xcode顶部选择你的iPhone作为目标
4. 点击运行

**首次真机运行需要**：
- Apple ID登录（免费账号即可）
- 自动签名：Signing & Capabilities → Automatically manage signing ✅
- Bundle Identifier修改为唯一的（如 `com.yourname.GameRec`）

## 🐛 可能遇到的问题

### 问题1: "Cannot find type 'Game' in scope"

**原因**：文件未正确添加到target

**解决**：
1. 选择报错的文件
2. 右侧面板 → File Inspector
3. 勾选 "Target Membership" → GameRec ✅

### 问题2: JSON文件找不到

**原因**：JSON未添加到Bundle Resources

**解决**：
1. 选择项目 → Build Phases
2. 展开 "Copy Bundle Resources"
3. 点击 + 添加3个JSON文件

### 问题3: 编译错误："Module compiled with Swift X cannot be imported"

**原因**：Swift版本不匹配

**解决**：
1. Build Settings → Swift Language Version → Swift 5
2. Clean Build Folder (⇧⌘K)
3. 重新Build (⌘B)

### 问题4: "Data/SeedData/games.json" not found

**原因**：JSON文件路径错误

**解决**：
1. 确保在Xcode中文件结构是：
   ```
   GameRec/
   └── Data/
       └── SeedData/
           ├── games.json
           ├── prices.json
           └── user_library.json
   ```
2. 修改DataStore.swift中的路径：
   ```swift
   Bundle.main.url(forResource: filename, 
                   withExtension: "json", 
                   subdirectory: "Data/SeedData")
   ```

## 💡 推荐工作流

1. **Windows**：写代码（Models、算法、UI）
2. **Git Push**：推送到GitHub
3. **Mac**：拉取代码，在Xcode编译运行
4. **iPhone**：真机测试
5. **反馈**：在Windows上继续修改

这样可以充分利用Windows的开发环境（VS Code）和Mac的编译环境（Xcode），完美配合！

---

## ⚠️ 注意事项

1. **不要提交xcodeproj到Git**（已在.gitignore）
   - 每个人的Xcode配置不同
   - 在Mac上重新创建即可

2. **JSON文件必须提交**
   - 种子数据是必需的
   - 确保编码为UTF-8

3. **及时提交代码**
   - 随时保存进度
   - 方便回滚和协作

4. **使用分支**（进阶）
   ```bash
   git checkout -b feature/new-ui
   # 开发新功能
   git push -u origin feature/new-ui
   ```
