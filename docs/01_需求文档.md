# GameRec 项目需求文档

> 版本：v0.1（Step1）
> 作者：数据分析师

---

##1. 项目背景

做一个 iOS 游戏推荐 App。区别于"内容帖子"型社区（什么 Steam社区、贴吧），本 App 的核心是**直接推荐游戏本身**。

##2. 用户场景

用户（游戏玩家）可以：
1.绑定多个游戏平台账号（PSN / Steam / Switch）
2. 系统读取该账号下已玩过的游戏 + 时长 +进度 +成就
3. 系统读取价格信息
4. 系统给出两层推荐：**类别 → 类内未玩游戏**
5. 用户可以"换一批"刷新类别或游戏

##3. 功能需求（MVP）

###3.1账号绑定
- 支持多个平台（PSN、Steam、Switch）账号并行绑定
- MVP阶段：手动录入形式（后续接口位置预留）
- 数据字段：账号名 + 该账号下已玩游戏列表（含时长/进度/成就）

###3.2 数据获取维度
|维度 | 来源 | MVP阶段 |
|------|------|---------|
| 已玩游戏列表 |手动录入 | ✅ JSON模拟 |
| 游戏时长 |手动录入 | ✅ JSON模拟 |
| 完成进度 |手动录入 | ✅ JSON模拟 |
|成就 |手动录入 | ✅ JSON模拟 |
| 价格 | 本地价格表 | ✅写死 |
| 价格折扣 | 本地价格表 | ✅写死 |

###3.3 推荐功能
- **第一层**：推荐3-5 个游戏类别（如 RPG、Roguelike、Galgame 等）
- **第二层**：选定类别后，推荐5-8款**该类别下没玩过**的游戏
- **换一批**：
 -类别换一批：刷新 TopN类别
 - 游戏换一批：在同一类别内换一批推荐

###3.4 多账号合并
-同一款游戏在多个平台都玩过 →视为已玩（去重）
- 时长累加
-进度取最大值

##4. 非功能需求

| 项 | 要求 |
|------|------|
|平台 | iOS15.0+ |
|性能 | 推荐计算 <200ms（本地） |
|离线 | 完全离线可用 |
|隐私 | 所有数据本地存储，不上传 |
|响应式 | 支持 iPhone 横竖屏 |

##5.范围之外（Out of Scope）

- ❌真实 API接入（PSN/Switch官方未开放）
- ❌ 多用户/账号系统
- ❌推送通知
- ❌ 游戏内购推荐
- ❌社交功能（关注、评论等）
- ❌ 安卓版本

##6. MVP验收标准

- [] 能绑定至少1 个平台账号（手动录入）
- [] 录入的游戏能在"我的游戏库"看到
- [] 首页能展示推荐类别 + 推荐游戏
- [] 类别换一批、游戏换一批能正常工作
- [] 推荐结果能展示推荐依据（可解释）
- [] Xcode模拟器运行流畅
