##2. 模块划分

###2.1 Models 层（纯数据结构）
不依赖任何业务逻辑，纯 Codable + Equatable。
- `Game.swift` - 游戏元数据
- `UserGameRecord.swift` - 用户已玩游戏记录
- `Platform.swift` -平台枚举（psn/steam/switch）
- `GameCategory.swift` -类别枚举（rpg/roguelike/galgame/...）
- `PriceRecord.swift` - 价格快照

###2.2 Data 层（存储）
负责 JSON持久化 +内存缓存。
- `DataStore.swift` -统一 JSON读写入口
- `GameRepository.swift` - 游戏库 CRUD
- `UserLibraryRepository.swift` - 用户库 CRUD（多账号合并）
- `PriceRepository.swift` - 价格 CRUD

###2.3 Services 层（业务服务）
- `AccountBindingService.swift` - 多账号管理（手动录入接口）
- `PriceService.swift` - 价格获取（MVP用本地表）
- `ImportService.swift` -批量导入游戏数据
- `RecommendationService.swift` - 推荐门面

###2.4 Recommendation 层（算法核心）⭐
- `FeatureExtractor.swift` - 游戏 tags → one-hot 向量
- `Scorer.swift` -类别/游戏评分公式
- `CategoryRecommender.swift` -类别推荐
- `GameRecommender.swift` - 类内游戏推荐
- `RefreshStrategy.swift` -换一批种子策略
- `RecommendationWeights.swift` -权重配置（数据分析师可调）

###2.5 Views 层（SwiftUI）
- `ContentView.swift` - Tab 主容器（首页/游戏库/账号）
- `HomeView.swift` - 推荐首页
- `LibraryView.swift` - 我的游戏库
- `BindAccountView.swift` -账号绑定表单
- `GameDetailView.swift` - 游戏详情
- `ImportDataView.swift` -批量录入
- `Components/` -复用组件（GameCard / CategoryChip / RefreshButton）

##3. 数据流

###3.1 推荐首页加载流程
```
HomeView.init
 ↓
HomeViewModel.refreshAll()
 ↓
RecommendationService.recommend()
 ├→ CategoryRecommender.recommend(用户库, 价格表) → [CategoryScore]
 └→ GameRecommender.recommend(类别, 用户库, 价格表) → [GameScore]
 ↓
@Published var state → UI 自动刷新
```

###3.2"换一批"流程
```
用户点击"类别换一批"
 ↓
HomeViewModel.refreshCategories(seed++ 用刷新次数)
 ↓
CategoryRecommender.recommend(seed=seed)
 └→内部偏移 +随机选择
 ↓
UI 更新
```

##4.关键设计决策

###4.1 多账号合并
- 数据结构：`UserGameRecord.gameId + platform` 作为联合唯一键
-合并逻辑：
 - 时长：累加
 -进度：取最大值
 -成就：取并集
- 在 `UserLibraryRepository.mergedLibrary()`暴露

###4.2 推荐可解释性
- 每个推荐结果都携带 `reason: String`
- 在 `Scorer` 里生成："因为你喜欢：开放世界、剧情向"
- UI 显示在卡片副标题

###4.3权重可调
- 所有权重集中在 `RecommendationWeights.swift`
- 数据分析师可在调试面板修改，运行时生效
-预留 UI：`DebugWeightsView`（Step7 实现）
