> 版本：v0.1

## 1. 设计理念

### 核心原则
- **清晰直观**：两层推荐结构一目了然
- **信息密度适中**：避免信息过载，突出重点
- **可解释性**：每个推荐都显示理由
- **操作便捷**："换一批"随手可得

### 视觉风格
- **现代简约**：iOS 原生设计语言
- **卡片式布局**：信息分组清晰
- **渐变色点缀**：蓝色系为主，橙色评分突出
- **圆角设计**：友好柔和的视觉体验

## 3. 交互流程

### 初始加载
1. 进入首页 → 显示骨架屏/加载指示器
2. 加载用户统计 → 显示统计卡片
3. 加载类别推荐（TopN=5）→ 显示类别卡片
4. 自动选中第一个类别
5. 加载该类别游戏（TopN=8）→ 显示游戏卡片

### 换一批流程

**类别换一批**：
```
用户点击"换一批" 
  ↓
seed++（0→1）
  ↓
重新请求类别推荐（seed=1）
  ↓
UI更新：显示下一批5个类别
  ↓
自动选中新的第一个类别
  ↓
加载新类别的游戏
```

**游戏换一批**：
```
用户点击游戏区"换一批"
  ↓
gameSeed++（0→1）
  ↓
重新请求当前类别游戏（seed=1）
  ↓
UI更新：显示下一批8个游戏
```

### 选择类别流程
```
用户点击类别卡片
  ↓
更新选中状态（高亮）
  ↓
重置gameSeed=0
  ↓
加载该类别游戏
  ↓
UI更新
```

### 下拉刷新
```
用户下拉
  ↓
重置所有seed=0
  ↓
重新加载类别
  ↓
重新加载游戏（如果有选中）
```

## 5. 色彩规范

### 主色调
- **主色**：系统蓝色 `Color.blue`
- **强调色**：橙色（评分）`Color.orange`
- **警示色**：红色（折扣）`Color.red`
- **次要色**：灰色 `Color.secondary`

### 背景色
- **卡片背景**：`Color(uiColor: .secondarySystemGroupedBackground)`
- **统计卡片**：`Color.blue.opacity(0.1)`
- **类别标签**：`Color.blue.opacity(0.2)`

### 文字色
- **主文字**：`.primary`（系统自适应）
- **次要文字**：`.secondary`
- **白色文字**：`.white`（徽章内）

## 7. 动画效果

### 推荐的动画
1. **卡片出现**：渐显 + 轻微缩放（0.95→1.0）
2. **选中类别**：轻微脉冲动画
3. **换一批**：淡出旧内容 → 淡入新内容
4. **下拉刷新**：系统默认刷新动画

### 实现方式
```swift
// 卡片出现动画
.transition(.scale.combined(with: .opacity))
.animation(.easeOut(duration: 0.3), value: recommendedCategories)

// 选中动画
.scaleEffect(isSelected ? 1.05 : 1.0)
.animation(.spring(response: 0.3), value: isSelected)
```

## 8. 响应式设计

### 适配iPhone
- **横屏**：类别卡片一行显示更多
- **竖屏**：标准布局
- **小屏幕**（SE）：缩小padding，减少统计卡片高度

### 适配iPad（预留）
- **Split View**：类别列表在左侧固定
- **Slide Over**：游戏卡片堆叠显示

## 9. 无障碍支持

### VoiceOver

.accessibilityValue("评分\(score)分")

### 图片加载
- 封面图使用异步加载（MVP阶段占位符）
- 实现图片缓存机制

### 数据缓存

- 避免重复网络请求
