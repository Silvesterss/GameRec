## 2. Swift Models

### 2.1 Platform（平台枚举）

**文件**：`GameRec/Models/Platform.swift`

```swift
enum Platform: String, Codable, CaseIterable {
    case psn = "PSN"
    case steam = "Steam"
    case nintendoSwitch = "Switch"
}
```

**说明**：支持的三个游戏平台

---

### 2.2 GameCategory（游戏类别枚举）

**文件**：`GameRec/Models/GameCategory.swift`

**支持的类别**（29种）：
- RPG、动作RPG
- Roguelike、Roguelite、银河恶魔城
- 射击、第一人称射击、第三人称射击
- 动作、冒险、解谜、平台跳跃
- 格斗、赛车、体育、模拟、策略
- MOBA、MMO、沙盒、生存、恐怖、潜行
- 音乐节奏、Galgame、卡牌
- 独立游戏、开放世界、魂系

**说明**：
- 每个类别都有 `baseWeight`（推荐权重基数）
- 热门类别（RPG/动作RPG/开放世界）权重为 1.2
- 中等类别（Galgame/独立游戏）权重为 1.0
- 其他类别权重为 1.1

### 2.3 Game（游戏元数据）

**文件**：`GameRec/Models/Game.swift`

**字段说明**：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | String | 全局唯一ID | "game_001" |
| title | String | 游戏名称 | "艾尔登法环" |
| categories | [GameCategory] | 所属类别（可多个） | ["动作RPG", "开放世界", "魂系"] |
| tags | [String] | 特征标签（用于相似度计算） | ["开放世界", "魂系", "困难"] |
| description | String | 游戏简介 | "FromSoftware的开放世界魂系游戏" |
| releaseYear | Int | 发行年份 | 2022 |
| platforms | [Platform] | 支持的平台 | ["PSN", "Steam"] |
| imageURL | String? | 封面图URL（MVP阶段为null） | null |

**计算属性**：
- `primaryCategory`: 主类别（取第一个）
- `isAvailableOn(_ platform)`: 是否支持指定平台
- `tagSet`: 标签集合（用于相似度计算）

**JSON 示例**：
```json
{
  "id": "game_001",
  "title": "艾尔登法环",
  "categories": ["动作RPG", "开放世界", "魂系"],
  "tags": ["开放世界", "魂系", "困难", "探索", "BOSS战"],
  "description": "FromSoftware与《冰与火之歌》作者合作的开放世界魂系游戏",
  "releaseYear": 2022,
  "platforms": ["PSN", "Steam"],
  "imageURL": null
}
```

### 2.4 UserGameRecord（用户游戏记录）

**文件**：`GameRec/Models/UserGameRecord.swift`

**字段说明**：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | String | 唯一ID：`{gameId}_{platform}` | "game_001_Steam" |
| gameId | String | 对应 Game.id | "game_001" |
| platform | Platform | 在哪个平台玩的 | "Steam" |
| hoursPlayed | Double | 游戏时长（小时） | 85.5 |
| progressPercentage | Int | 完成进度（0-100） | 78 |
| achievementsUnlocked | Int | 已解锁成就数 | 32 |
| totalAchievements | Int | 总成就数 | 42 |
| lastPlayedDate | Date? | 最后游玩时间（可选） | "2026-05-03T10:14:14" |

**计算属性**：
- `achievementCompletionRate`: 成就完成度（0.0-1.0）
- `isDeepEngagement`: 是否深度游玩（时长>20h 或 进度>50%）
- `isCompleted`: 是否已完成（进度≥100%）
- `engagementScore`: 投入强度评分（0-10），综合时长、进度、成就

**JSON 示例**：
```json
{
  "id": "game_001_Steam",
  "gameId": "game_001",
  "platform": "Steam",
  "hoursPlayed": 85.5,
  "progressPercentage": 78,
  "achievementsUnlocked": 32,
  "totalAchievements": 42,
  "lastPlayedDate": "2026-05-03T10:14:14.730914"
}
```

### 2.5 PriceRecord（价格记录）

**文件**：`GameRec/Models/PriceRecord.swift`

**字段说明**：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | String | 唯一ID：`{gameId}_{platform}` | "game_001_Steam" |
| gameId | String | 对应 Game.id | "game_001" |
| platform | Platform | 平台 | "Steam" |
| originalPrice | Double | 原价（CNY） | 298.0 |
| currentPrice | Double | 当前价格（CNY） | 178.8 |
| discountPercentage | Int | 折扣百分比（0-100） | 40 |
| isFree | Bool | 是否免费游戏 | false |
| lastUpdated | Date | 价格更新时间 | "2026-06-13T10:14:14" |

**计算属性**：
- `isOnSale`: 是否正在打折
- `savedAmount`: 节省的金额
- `priceFriendlinessScore`: 价格友好度评分（0-10）
  - 免费游戏 = 10分
  - 折扣分（最高5分）+ 价格分（最高5分）
  - 100元以下=5分，200元=3分，300元以上=1分
- `formattedPrice`: 格式化价格字符串（如 "¥178"）
- `formattedDiscount`: 格式化折扣字符串（如 "-40%"）

**JSON 示例**：
```json
{
  "id": "game_001_Steam",
  "gameId": "game_001",
  "platform": "Steam",
  "originalPrice": 298.0,
  "currentPrice": 178.8,
  "discountPercentage": 40,
  "isFree": false,
  "lastUpdated": "2026-06-13T10:14:14.730914"
}
```

## 3. 种子数据文件

### 3.1 games.json

**位置**：`GameRec/Data/SeedData/games.json`

**说明**：
- 包含 120 款游戏的元数据
- 包括 22 款真实游戏（艾尔登法环、女神异闻录5 等）
- 补充 98 款虚构游戏用于测试

**统计**：
- 总游戏数：120
- 类别覆盖：29 种
- 平台分布：PSN、Steam、Switch

### 3.2 prices.json

**位置**：`GameRec/Data/SeedData/prices.json`

**说明**：
- 239 条价格记录（每个游戏在多个平台上有不同价格）
- 30% 的游戏有折扣（20%-60% off）
- 价格范围：免费 ~ ¥468

**价格分布**：
- 免费游戏：约 5%
- ¥40-100：约 40%
- ¥100-300：约 45%
- ¥300+：约 10%

### 3.3 user_library.json

**位置**：`GameRec/Data/SeedData/user_library.json`

**说明**：
- 35 条用户已玩游戏记录
- 模拟一个中度玩家的游戏库
- 游戏时长范围：5-150 小时
- 完成进度范围：10%-100%

**统计**：
- 深度游玩游戏（>20h）：约 60%
- 已完成游戏（100%）：约 20%
- 平台分布：随机

## 4. 数据关联关系

```
Game (游戏元数据)
  ├─ id: "game_001"
  │
  ├─ PriceRecord (价格)
  │   ├─ id: "game_001_PSN"
  │   ├─ id: "game_001_Steam"
  │   └─ gameId: "game_001" ← 关联
  │
  └─ UserGameRecord (用户记录)
      ├─ id: "game_001_Steam"
      └─ gameId: "game_001" ← 关联
```

**关联逻辑**：
1. **Game** 是核心数据，通过 `id` 唯一标识
2. **PriceRecord** 通过 `gameId` 关联到 Game，每个游戏在每个平台有一条价格记录
3. **UserGameRecord** 通过 `gameId` 关联到 Game，记录用户在某个平台上的游玩数据
4. **多账号合并**：同一 `gameId` 在不同 `platform` 的记录视为已玩（去重）

## 5. 数据生成脚本

**文件**：`scripts/generate_seed_data.py`

**功能**：
- 生成 120 款游戏库
- 生成 239 条价格记录
- 生成 35 条用户游戏记录

**使用方法**：
```bash
python scripts/generate_seed_data.py
```

**配置项**：
- `NUM_GAMES = 120`：生成的游戏数量
- `NUM_USER_PLAYED = 35`：用户已玩的游戏数量
- `random.seed(42)`：随机种子（可复现）

## 6. 使用示例

### 6.1 在 Swift 中加载 JSON

```swift
// 加载游戏库
if let url = Bundle.main.url(forResource: "games", withExtension: "json", subdirectory: "Data/SeedData") {
    let data = try Data(contentsOf: url)
    let games = try JSONDecoder().decode([Game].self, from: data)
}

// 加载用户库
if let url = Bundle.main.url(forResource: "user_library", withExtension: "json", subdirectory: "Data/SeedData") {
    let data = try Data(contentsOf: url)
    let records = try JSONDecoder().decode([UserGameRecord].self, from: data)
}

// 加载价格表
if let url = Bundle.main.url(forResource: "prices", withExtension: "json", subdirectory: "Data/SeedData") {
    let data = try Data(contentsOf: url)
    let prices = try JSONDecoder().decode([PriceRecord].self, from: data)
}
```

### 6.2 多账号合并示例

```swift
// 用户在 Steam 玩了 50 小时
let record1 = UserGameRecord(gameId: "game_001", platform: .steam, hoursPlayed: 50, ...)

// 用户在 PSN 玩了 30 小时
let record2 = UserGameRecord(gameId: "game_001", platform: .psn, hoursPlayed: 30, ...)

// 合并逻辑：
// - 时长累加：50 + 30 = 80 小时
// - 进度取最大值
// - 成就取并集
```

## 8. 注意事项

1. **日期格式**：所有日期使用 ISO 8601 格式（`yyyy-MM-dd'T'HH:mm:ss`）
2. **价格单位**：统一使用 CNY（人民币）
3. **ID 规则**：
   - 游戏 ID：`game_XXX`（3位数字）
   - 价格/记录 ID：`{gameId}_{platform}`
4. **数据完整性**：所有 `gameId` 必须在 `games.json` 中存在
5. **MVP 限制**：`imageURL` 字段暂时为 `null`，v1.1 后补充
